Dieses Thema beschreibt die PowerShell-Cmdlets aus der Vorschauversion von Azure PowerShell 1.0. Wenn Sie diese Vorschauversion jetzt nicht installieren möchten, können Sie weiterhin mit Azure PowerShell 0.9.8 arbeiten. In den meisten Fällen ist der einzige Unterschied zwischen den beiden Versionen, dass die Cmdlet-Namen der Vorschau für Version 1.0 das Muster {Verb}-AzureRm{Nomen} haben, während der Name der Version 0.9.8 nicht **Rm** enthält (z. B. "New-AzureRmResourceGroup" statt "New-AzureResourceGroup"). Wenn der Unterschied zwischen den Versionen größer ist, werden in diesem Thema Beispiele für beide Versionen aufgeführt.

Wenn Sie Azure PowerShell 0.9.8 verwenden, müssen Sie zunächst den Ressourcen-Manager-Modus aktivieren, indem Sie den Befehl **Switch-AzureMode AzureResourceManager** ausführen. Dieser Befehl ist in der Vorschau für 1.0 nicht erforderlich.

Informationen zur Vorschau für Version 1.0, einschließlich Verwendungsempfehlungen, finden Sie unter [Vorschau für Azure PowerShell 1.0](https://azure.microsoft.com/blog/azps-1-0-pre/). Informationen zur Installation der Vorschauversion von Azure PowerShell 1.0 finden Sie unter [Cmdlets für Azure-Ressourcen-Manager](https://msdn.microsoft.com/library/mt125356.aspx). Informationen zu wichtigen Änderungen bei den Ressourcen-Manager-Befehlen finden Sie unter [Änderungen der PowerShell-Cmdlets für die Verwaltung in Azure-Ressourcen-Manager](../articles/powershell-preview-resource-manager-changes.md).
