

Bilder werden in Azure verwendet, um einen neuen virtuellen Computer mit einem Betriebssystem bereitzustellen. Ein Bild möglicherweise auch eine oder mehrere Datenträger für Daten. Images stehen aus verschiedenen Quellen zur Verfügung:

* Azure bietet Images in den [Marketplace](https://azure.microsoft.com/gallery/virtual-machines/). Es gibt neueren Versionen von Windows Server und Distributionen des Linux-Betriebssystems. Einige Images umfassen außerdem Anwendungen wie SQL Server. Abonnenten von MSDN Benefit und MSDN nutzungsbasierte haben Zugriff auf Weitere Images.
* Die open-Source-Community bietet Images über [VM-Depot](http://vmdepot.msopentech.com/List/Index).
* Sie können auch speichern und verwenden Ihre eigenen Images in Azure, indem Sie entweder einen vorhandenen Azure virtuellen Computer für die Verwendung als Image erfassen oder ein Image hochladen.

## <a name="about-vm-images-and-os-images"></a>Informationen zu VM-Images und Betriebssystemimages
Zwei Typen von Bildern in Azure verwendet werden können: *VM-Image* und *Betriebssystemimage*. Ein VM-Image enthält ein Betriebssystem installiert ist und alle Datenträger, die an einen virtuellen Computer angefügt werden, wenn das Image erstellt wird. Ein VM-Image ist der neuere Typ des Bilds. Vor der Einführung von VM-Images, möglicherweise ein Image in Azure nur ein generalisiertes Betriebssystem und keine zusätzlichen Datenträger. Ein VM-Image, das nur ein generalisiertes Betriebssystem enthält entspricht im Grunde den ursprünglichen Typ des Bilds, das Betriebssystemabbild.

Sie können eigene Images basierend auf einem virtuellen Computer in Azure oder an anderer Stelle ausgeführter virtueller Computer erstellen, die Sie kopieren und hochladen. Wenn Sie ein Bild verwenden, um mehr als eine virtuelle Maschine erstellen möchten, müssen Sie für die Verwendung als Image vorbereiten, indem Sie ihn generalisieren. Um ein Windows Server-Image zu erstellen, führen Sie den Sysprep-Befehl auf dem Server, um es zu generalisieren, bevor Sie die VHD-Datei hochladen. Ausführliche Informationen zu Sysprep finden Sie unter [mithilfe von Sysprep: Einführung](http://go.microsoft.com/fwlink/p/?LinkId=392030) und [Sysprep-Unterstützung für Serverrollen](https://msdn.microsoft.com/windows/hardware/commercialize/manufacture/desktop/sysprep-support-for-server-roles). Sichern Sie den virtuellen Computer vor dem Ausführen von Sysprep aus. Erstellen eine Linux-Image hängt von der Verteilung von. In der Regel müssen Sie eine Reihe von Befehlen ausgeführt, die spezifisch für die Verteilung sind, und führen Sie den Azure Linux-Agent.
