Azure bietet die folgenden integrierten RBAC-Rollen für den Zugriff auf Speicherdaten:

- [Besitzer von Speicherblobdaten (Vorschau):](../articles/role-based-access-control/built-in-roles.md#storage-blob-data-owner-preview) Dient zum Festlegen des Besitzes sowie zum Verwalten der POSIX-Zugriffssteuerung für Azure Data Lake Storage Gen2 (Vorschauversion). Weitere Informationen finden Sie unter [Zugriffssteuerung in Azure Data Lake Storage Gen2](../articles/storage/blobs/data-lake-storage-access-control.md).
- [Mitwirkender an Storage-Blobdaten (Vorschau):](../articles/role-based-access-control/built-in-roles.md#storage-blob-data-contributor-preview) Dient zum Gewähren von Lese-/Schreib-/Löschberechtigungen für Blobspeicherressourcen.
- [Storage-Blobdatenleser (Vorschau):](../articles/role-based-access-control/built-in-roles.md#storage-blob-data-reader-preview) Dient zum Gewähren von Leseberechtigungen für Blobspeicherressourcen.
- [Mitwirkender an Storage-Warteschlangendaten (Vorschau):](../articles/role-based-access-control/built-in-roles.md#storage-queue-data-contributor-preview) Dient zum Gewähren von Lese-/Schreib-/Löschberechtigungen für Azure-Warteschlangen.
- [Storage-Warteschlangendatenleser (Vorschau):](../articles/role-based-access-control/built-in-roles.md#storage-queue-data-reader-preview) Dient zum Gewähren von Leseberechtigungen für Azure-Warteschlangen.

Weitere Informationen dazu, wie integrierte Rollen für Azure Storage definiert sind, finden Sie unter [Grundlegendes zu Rollendefinitionen](../articles/role-based-access-control/role-definitions.md#management-and-data-operations-preview). Weitere Informationen zum Erstellen benutzerdefinierter RBAC-Rollen finden Sie unter [Erstellen benutzerdefinierter Rollen für die rollenbasierte Zugriffssteuerung in Azure](../articles/role-based-access-control/custom-roles.md).
