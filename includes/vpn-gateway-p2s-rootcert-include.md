Sie können entweder ein Stammzertifikat verwenden, das mit einer Unternehmenslösung generiert wurde (empfohlen), oder ein selbstsigniertes Zertifikat generieren. Führen Sie bei der Verwendung eines selbstsignierten Zertifikats die Schritte im Artikel [Erstellen eines selbstsignierten Stammzertifikats für Punkt-zu-Standort-Verbindungen](../articles/vpn-gateway/vpn-gateway-certificates-point-to-site.md#rootcert) aus. In diesen Schritten kommen spezifische Einstellungen zum Einsatz, die zum Generieren eines P2S-kompatiblen Zertifikats erforderlich sind.

Nach dem Erstellen des Stammzertifikats exportieren Sie die Daten des öffentlichen Zertifikats (nicht den privaten Schlüssel) als Base64-codierte X.509-CER-Datei. Anschließend laden Sie die Daten des öffentlichen Zertifikats aus dem Stammzertifikat nach Azure hoch.

* **Unternehmenszertifikat:** Bei einer Unternehmenslösung können Sie Ihre vorhandene Zertifikatkette verwenden. Rufen Sie die CER-Datei für das Stammzertifikat ab, das Sie verwenden möchten.
* **Selbstsigniertes Stammzertifikat:** Wenn Sie keine Unternehmenszertifikatlösung verwenden, müssen Sie ein selbstsigniertes Stammzertifikat generieren. Das Stammzertifikat muss bestimmte Werte enthalten, damit es mit einer Punkt-zu-Standort-Verbindung verwendet werden kann. Anweisungen finden Sie in folgenden Artikeln:
  * Informationen zum Erstellen eines selbstsigniertes Stammzertifikats finden Sie unter [Erstellen eines selbstsignierten Stammzertifikats für Punkt-zu-Standort-Verbindungen](../articles/vpn-gateway/vpn-gateway-certificates-point-to-site.md#rootcert).
  * Informationen zum Exportieren des öffentlichen Schlüssels (CER-Datei) finden Sie unter [Exportieren des öffentlichen Schlüssels](../articles/vpn-gateway/vpn-gateway-certificates-point-to-site.md#cer).
