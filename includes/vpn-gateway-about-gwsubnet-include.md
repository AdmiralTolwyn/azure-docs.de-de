Das Gateway für virtuelle Netzwerke verwendet ein bestimmtes Subnetz, das als „GatewaySubnet“ bezeichnet wird. Das Gatewaysubnetz enthält die IP-Adressen, die von den VPN-Gatewaydiensten verwendet werden. Wenn Sie ein Gatewaysubnetz erstellen, muss es den Namen „GatewaySubnet“ erhalten.  Wenn Sie einem Subnetz den Namen „GatewaySubnet“ zuweisen, teilen Sie Azure mit, wo die Gatewaydienste erstellt werden sollen. Wenn Sie einen anderen Namen wählen, tritt bei der Konfiguration des VPN-Gateways ein Fehler auf.

Die IP-Adressen in „GatewaySubnet“ werden den Gatewaydiensten zugeordnet. Wenn Sie „GatewaySubnet“ erstellen, geben Sie die Anzahl der im Subnetz enthaltenen IP-Adressen an. Die Größe des von Ihnen angegebenen „GatewaySubnet“ richtet sich nach der VPN-Gatewaykonfiguration, die Sie erstellen möchten. Es ist zwar möglich, „GatewaySubnet“ mit einer Größe von nur /29 zu erstellen, es wird jedoch empfohlen, /27 oder /28 auszuwählen, um ein größeres Subnetz mit mehr Adressen zu erstellen. Die Verwendung eines größeren Gatewaysubnetzes ermöglicht die Vergabe einer ausreichenden Zahl von IP-Adressen für potenzielle zukünftige Konfigurationen.
