# Was ist Azure Active Directory?

Azure Active Directory ist ein Dienst, der Funktionen zur Identitäts- und Zugriffsverwaltung in der Cloud bietet. Active Directory ist ein Dienst, den Kunden für lokale Identitätsverwaltung über das Windows Server-Betriebssystem nutzen können. Ähnlich dazu ist Azure Active Directory (Azure AD) ein Dienst, mit dem Sie über Azure cloudbasierte Identitätsverwaltung durchführen können. [Weitere Informationen][Weitere Informationen]

Da es sich um das Cloudverzeichnis Ihres Unternehmens handelt, entscheiden Sie, welche Benutzer es gibt, welche Informationen in der Cloud gespeichert werden, wer diese Informationen nutzen oder verwalten kann und welche Anwendungen oder Dienste auf die Informationen zugreifen dürfen.

Wenn Sie Azure AD verwenden, ist es die Aufgabe von Microsoft, Active Directory in der Cloud bereitzustellen und eine hohe Skalierbarkeit, Hochverfügbarkeit und integrierte Notfallwiederherstellung zu bieten sowie Ihre Anforderungen an Vertraulichkeit und Sicherheit der Informationen Ihres Unternehmens zu erfüllen.

### Integration Ihres lokalen Active Directory

Sie können Azure AD als eigenständiges Cloudverzeichnis für Ihr Unternehmen verwenden. Es ist auch möglich, ein vorhandenes lokales Active Directory in Azure AD zu integrieren. Einige Integrationsfunktionen umfassen Verzeichnissynchronisierung und Single-Sign-On. So erhalten lokale Identitäten mehr Möglichkeiten in der Cloud, und es entsteht eine optimierte Benutzerfreundlichkeit für Administratoren und Benutzer.
 [Weitere Informationen][1]

### Integration Ihrer Anwendungen

Anwendungsentwickler können ihre Anwendungen in Azure AD integrieren, um ihren Nutzern die Funktion Single-Sign-On zu bieten. So können Unternehmensanwendungen in der Cloud gehostet werden, und Benutzer können sich ganz einfach mit den Anmeldeinformationen ihres Unternehmens authentifizieren. Außerdem können Anbieter von Software-as-a-Service (SaaS) einfachere Authentifizierung bieten, wenn sich Benutzer in Unternehmen mit Azure AD bei ihren Diensten authentifizieren. Entwickler können auch die Graph-API verwenden, um Verzeichnisdaten für Verwaltungsentitäten abzurufen, beispielsweise für Benutzer oder Gruppen. [Weitere Informationen][2]
