Aktivieren der Diagnose in Azure
================================

Mit der Azure-Diagnose können Sie Diagnosedaten von einer Worker- oder Webrolle erfassen, die in Azure ausgeführt wird. Die Diagnosedaten können zum Debuggen und Beheben von Fehlern, Messen der Leistung, Überwachen der Ressourcenauslastung, Analysieren des Datenverkehrs, Planen der Kapazität und Durchführen der Überwachung verwendet werden.

Sie können die Diagnose mithilfe des Azure SDK 2.0 oder höher vor der Bereitstellung oder zur Laufzeit in Visual Studio 2012 oder Visual Studio 2013 konfigurieren. Weitere Informationen finden Sie unter [Konfigurieren der Azure-Diagnose](http://msdn.microsoft.com/en-us/library/windowsazure/dn186185.aspx).

Weitere ausführliche Anweisungen zum Erstellen einer Protokollierungs- und Nachverfolgungsstrategie und Verwenden von Diagnose und anderen Methoden zum Beheben von Problemen finden Sie unter [Bewährte Methoden zur Problembehandlung bei der Entwicklung von Azure-Anwendungen](http://msdn.microsoft.com/en-us/library/windowsazure/hh771389.aspx).

Informationen zur manuellen Konfiguration der Diagnose in Ihrer Anwendung oder Remoteänderung der Diagnosemonitorkonfiguration finden Sie unter [Sammeln von Protokollierungsdaten mit der Azure-Diagnose](http://msdn.microsoft.com/en-us/library/windowsazure/gg433048.aspx).

-   [Remoteänderung der Diagnosemonitorkonfiguration](http://msdn.microsoft.com/en-us/library/windowsazure/gg432992.aspx): Nach der Bereitstellung der Anwendung können Sie die Diagnosekonfiguration ändern.
