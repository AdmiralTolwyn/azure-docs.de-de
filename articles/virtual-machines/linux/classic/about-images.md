# <a name="about-images-for-linux-virtual-machines"></a>Informationen zu Images für virtuelle Linux-Computer
> [!IMPORTANT]
> Azure verfügt über zwei verschiedene Bereitstellungsmodelle für das Erstellen und Verwenden von Ressourcen: [Resource Manager- und klassische Bereitstellung](../../../resource-manager-deployment-model.md). Dieser Artikel befasst sich mit der Verwendung des klassischen Bereitstellungsmodells. Microsoft empfiehlt für die meisten neuen Bereitstellungen die Verwendung des Ressourcen-Manager-Modells. Informationen zur Verwendung des Resource Manager-Modells durch Images finden Sie [hier](../../virtual-machines-linux-cli-ps-findimage.md?toc=%2fazure%2fvirtual-machines%2flinux%2ftoc.json).

[!INCLUDE [virtual-machines-common-classic-about-images](../../../../includes/virtual-machines-common-classic-about-images.md)]

## <a name="working-with-images"></a>Arbeiten mit Images
Die Azure-Befehlszeilenschnittstelle (CLI) für Mac, Linux und Windows dient zum Verwalten der Images, die in Ihrem Azure-Abonnement zur Verfügung stehen. Sie können für einige Aufgaben für Images auch das Azure-Portal nutzen, doch die Befehlszeile bieten Ihnen mehr Optionen.

Beispiele für die Verwendung der Tools finden Sie unter [Allgemeine Befehle der Azure-Befehlszeilenschnittstelle für Linux und Mac](../../virtual-machines-linux-cli-manage.md?toc=%2fazure%2fvirtual-machines%2flinux%2ftoc.json).
