# Testen Ihres Angebots einer Lösungsvorlage in der Stagingumgebung
In der Stagingumgebung wird Ihr Angebot in einem privaten "Sandkasten" bereitgestellt, in dem Sie es testen und seine Funktionalität vor der Veröffentlichung überprüfen können. Das Angebot wird in der Stagingumgebung genau wie bei einem Kunden angezeigt, der es bereitgestellt hat. Ihr Angebot **muss für die Freigabe in der Stagingumgebung zertifiziert sein**.

Sobald das Angebot in der Stagingphase ist, können Sie es im [Azure-Vorschauportal](https://ms.portal.azure.com/) anzeigen und testen.

Führen Sie die folgenden Schritte aus, um Ihr Angebot in der Stagingumgebung freizugeben und die Tests im [Azure-Vorschauportal](https://ms.portal.azure.com/) auszuführen.

1.	Navigieren Sie zum [Veröffentlichungsportal](https://publish.windowsazure.com). Wählen auf der Registerkarte **Lösungsvorlagen** Ihr Angebot und dann **Veröffentlichen** aus. Klicken Sie auf **Für Stagingumgebung freigeben**.
2.	Geben Sie die Liste der Azure-Abonnements an, mit denen Sie Ihr Angebot testen möchten.
3.	Melden Sie sich am Azure-Vorschauportal mit der Abonnement-ID an, mit der Sie Ihr Angebot im vorherigen Schritt in der Stagingumgebung freigegeben haben.
4.	Testen Sie Ihr Angebot im Azure-Vorschauportal auf die folgenden genannten Punkte.
  -	Marketinginhalte werden im Katalog ordnungsgemäß angezeigt.
  -	Bereitstellung der Topologie von A bis Z.
  -	Führen Sie Leistungs- und Belastungstests durch.
  -	Stellen Sie sicher, dass Ihre Topologie den bewährten Methoden entspricht.

## Nächster Schritt
Wenn Sie mit den Ergebnissen zufrieden sind, können Sie mit der letzten Phase beim Veröffentlichen Ihres Angebots fortfahren: **Schritt 4**, [Bereitstellen Ihres Angebots in Marketplace](marketplace-publishing-push-to-production.md). Nehmen Sie andernfalls Änderungen an Ihrem Angebot vor, und fordern Sie die Zertifizierung erneut an.

> [AZURE.NOTE]Für Änderungen an Marketinginhalten ist keine Zertifizierung erforderlich.
