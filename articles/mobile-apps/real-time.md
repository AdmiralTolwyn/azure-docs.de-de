# <a name="build-real-time-applications-with-azure-signalr"></a>Erstellen von Echtzeitanwendungen mit Azure SignalR

Mit dem [Azure SignalR](https://azure.microsoft.com/services/signalr-service/)-Dienst können Sie Anwendungen ganz einfach Echtzeitfunktionalität hinzufügen. Aufgrund dieser Echtzeitfunktionalität kann der Dienst Inhaltsupdates per Pushvorgang an verbundene mobile Anwendungen senden. Das Ergebnis ist, dass Apps aktualisiert werden, ohne dass der Server abgefragt werden muss oder neue HTTP-Anforderungen für Updates gesendet werden müssen.

## <a name="azure-signalr-features"></a>Azure SignalR-Features
- **Unterstützung für viele Clients**: SignalR funktioniert mit einer breiten Palette von Clients, z. B. Web- und mobile Browser, Desktopanwendungen, Serverprozesse, IoT-Geräte und Spielekonsolen.
- Der SignalR-Dienst unterstützt das **Zusammenarbeiten mehrerer Instanzen**, um auf Millionen von Verbindungen für mobile Anwendungen skalieren zu können. Der Dienst unterstützt außerdem mehrere globale Regionen für Sharding, Hochverfügbarkeit oder Notfallwiederherstellung.
- **Bereitstellen von umfangreichen APIs für unterschiedliche Benachrichtigungsmuster**, um Nachrichten an eine bestimmte Verbindung, an alle Verbindungen oder an eine Teilmenge von Verbindungen zu senden, die zu einem bestimmten Benutzer gehören oder in einer beliebigen Gruppe platziert wurden.
