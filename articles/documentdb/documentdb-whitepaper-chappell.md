# Introducing DocumentDB – Eine NoSQL Datenbank für Microsoft Azure
> [!NOTE]
> Dieses Whitepaper wurde zuletzt 2015 aktualisiert. DocumentDB-Updates wie die [globale Datenverteilung](documentdb-distribute-data-globally.md) und [partitionierte Sammlungen](documentdb-partition-data.md) sind hier noch nicht berücksichtigt. Eine aktuelle Einführung in DocumentDB finden Sie unter [Einführung in DocumentDB: JSON-NoSQL-Datenbanken](documentdb-introduction.md).
> 
> 

Sie sind sich nicht sicher, ob DocumentDB für Ihre Anwendung oder Daten geeignet ist? Möchten Sie erfahren, wie JSON-Daten gespeichert werden und wie der Zugriff auf diese Daten in Azure DocumentDB erfolgt? Wenn dies der Fall ist, ist dieses einführende Whitepaper von David Chappell ein guter Ausgangspunkt. Es wird erläutert, wie Sie von der Flexibilität profitieren können, die schemafreie JSON-Daten bieten und wie sie so mühelos mit Änderungen an der Datenstruktur verfahren können.

Dieses Whitepaper führt Sie auch durch die folgenden Aspekte:

* Verwenden des DocumentDB-Datenmodells, um JSON-Daten in Dokumente und Auflistungen innerhalb einer Datenbank zu speichern.
* Arbeiten mit Ihren Daten mithilfe von Clientbibliotheken, der RESTful-Zugriffsmethoden oder DocumentDB-SQL.
* Ausführen von Logik von der Datenbank durch Schreiben von gespeicherten Prozeduren (gespeicherte Prozeduren), Triggern oder benutzerdefinierten Funktionen (UDFs) und deren Speicherung in der Auflistung.
* Auswählen einer Konsistenz-Option, um die Zuverlässigkeits- und Leistungsanforderungen Ihrer Anwendung zu erfüllen.
* Auswählen einer Preisoption entsprechend Ihrer Durchsatzanforderungen.
