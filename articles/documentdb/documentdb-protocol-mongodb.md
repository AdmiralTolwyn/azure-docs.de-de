# DocumentDB-Protokollunterstützung für MongoDB

## Was ist Azure DocumentDB?
Bei Azure DocumentDB handelt es sich um einen vollständig verwalteten NoSQL-Datenbankdienst, der für hohe und vorhersagbare Leistung, hohe Verfügbarkeit, automatische Skalierung und einfache Bereitstellung entworfen wurde. Dank seines flexiblen Datenmodells, der durchgängig niedrigen Latenz und der umfangreichen Abfragefunktionen eignet sich der Dienst hervorragend für webbasierte und mobile Anwendungen, Spiele und IoT-Anwendungen sowie zahlreiche andere Anwendungen, für die eine nahtlose Skalierung erforderlich ist. Erfahren Sie mehr in der [Einführung in DocumentDB](documentdb-introduction.md).

## Was ist DocumentDB-Protokollunterstützung für MongoDB?
DocumentDB-Datenbanken können jetzt als Datenspeicher für Apps verwendet werden, die für MongoDB geschrieben wurden. Mithilfe der vorhandenen [Treiber](https://docs.mongodb.org/ecosystem/drivers/) für MongoDB können Anwendungen einfach und transparent mit DocumentDB kommunizieren. In vielen Fällen muss lediglich die Verbindungszeichenfolge geändert werden. Mithilfe dieser Vorschaufunktionalität können Kunden unter Verwendung der vollständig verwalteten und skalierbaren NoSQL-Datenbanken von DocumentDB leicht Anwendungen in der Azure-Cloud erstellen und ausführen und weiterhin die vorhandenen Kenntnisse und Tools für MongoDB verwenden.

Die DocumentDB-Protokollunterstützung für MongoDB ermöglicht die MongoDB-API-Kernfunktionen zum Erstellen, Lesen, Aktualisieren und Löschen (Create, Read, Update, Delete, CRUD) von Daten sowie Datenbankabfragen. Die derzeit implementierten Funktionen wurden basierend auf den Anforderungen gängiger Plattformen, Frameworks, Tools und wichtiger MongoDB-Kunden, die Azure für ihre Cloudplattform evaluieren, priorisiert.
