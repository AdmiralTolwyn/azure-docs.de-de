# Ihre Anwendung im Azure Active Directory-Anwendungskatalog auflisten

Um eine Anwendung im [Azure AD-Katalog](http://azure.microsoft.com/marketplace/active-directory/all/) und dem Azure Marketplace aufzulisten, die einmaliges Anmelden mit Azure Active Directory unterstützt, muss das Azure AD-Team die Integration überprüfen. Dafür senden Sie folgende Informationen an <waadpartners@microsoft.com>:

- Benutzername und Kennwort eines Testbenutzerkontos in einer Testinstanz von Azure Active Directory, mit der eine Anmeldung in Ihre Anwendung möglich ist.
- URL-Adresse und/oder Anweisungen, wie sich dieser Testbenutzer in die Anwendung anmelden sollte.
- Geben Sie eine kurze Beschreibung, wie Ihre Kunden die Verbindung zwischen der Anwendung und Azure AD herstellen. Dies könnte sein:
  - In die Anwendung mithilfe des Consent-Framework von Azure AD integriert.
  - Durch Konfigurieren von Azure AD als Identitätsanbieter (SAML 2.0, WS-Verbund oder OpenID Connect) mithilfe des administrativen Portals der Anwendung.
  - Durch Kontaktieren einer Person in Ihrer Organisation, um die Verbindung manuell herzustellen.
- Wenn Ihre Anwendung die Konfiguration von Azure AD als Identitätsanbieter über ein administratives Self-Service-Portal unterstützt, dann geben Sie bitte auch ein Test-Administratorkonto für die Überprüfung an.

Nach Abschluss der Tests kann die Anwendung im Azure Active Directory-Anwendungskatalog und im Azure Marketplace aufgelistet werden. Senden Sie die Antworten auf folgende Fragen an <waadpartners@microsoft.com>.

>
> Haben Sie Ihre Anwendung in Azure AD integriert und getestet (siehe [Authentifizierungsszenarien für Azure Active Directory](active-directory-authentication-scenarios.md))?
>
> Client-ID der Anwendung, wie in den Anwendungsdetails auf https://manage.windowsazure.com gezeigt:
>

>
