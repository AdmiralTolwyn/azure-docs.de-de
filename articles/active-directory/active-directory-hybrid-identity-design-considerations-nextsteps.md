#Überlegungen zum Entwurf der Azure Active Directory-Hybrid-Identität – Nächste Schritte
Nachdem Sie die Anforderungen definiert und alle Optionen für Ihre Lösung zur Verwaltung mobiler Geräte (Mobile Device Management, MDM) untersucht haben, können Sie die nächsten Schritte zur Bereitstellung der unterstützenden Infrastruktur ausführen, die für Sie und Ihr Unternehmen geeignet ist.

##Hybrid-Identitätslösungen
Die Nutzung spezieller Lösungsszenarien, die für Ihre Anforderungen geeignet sind, ist eine gute Möglichkeit, die Bereitstellung einer Infrastruktur für die Verwaltung mobiler Geräte zu prüfen und die damit verbundenen Details zu planen. Die folgenden Lösungen verdeutlichen die häufigsten Szenarien für die Verwaltung mobiler Geräte:

- Unter [Lösung: Verwalten mobiler Geräte und PCs in Unternehmensumgebungen](https://technet.microsoft.com/de-DE/library/dn582037.aspx) erhalten Sie Informationen, wie Sie mobile Geräte verwalten, indem Sie Ihre lokale System Center 2012 Configuration Manager-Infrastruktur mit Microsoft Intune in die Cloud erweitern. Diese Hybrid-Infrastruktur dient IT-Spezialisten in mittleren und großen Unternehmen als Unterstützung bei der Einrichtung von BYOD und des Remotezugriffs, während gleichzeitig der Verwaltungsaufwand reduziert wird.
- Unter [Lösung: Verwalten mobiler Geräte für Configuration Manager 2007](https://technet.microsoft.com/de-DE/library/dn508400.aspx) erhalten Sie Informationen zur Verwaltung mobiler Geräte, wenn Ihre Infrastruktur auf System Center Configuration Manager 2007 basiert. Bei dieser Lösung wird Ihnen gezeigt, wie Sie einen einzelnen Server mit System Center 2012 Configuration Manager einrichten, damit Sie dann Microsoft Intune ausführen und die Vorteile in Bezug auf die Verwaltung mobiler Geräte nutzen können, die diese Anwendung bietet.
- Unter [Lösung: Verwalten mobiler Geräte in kleinen Umgebungen](https://technet.microsoft.com/de-DE/library/dn715906.aspx) erhalten Sie Informationen für kleine Unternehmen, die Unterstützung für die Verwaltung mobiler Geräte benötigen. Es wird erläutert, wie Sie Microsoft Intune zum Erweitern Ihrer aktuellen Infrastruktur verwenden, um die Verwaltung mobiler Geräte und BYOD zu unterstützen. Bei dieser Lösung wird das einfachste Szenario beschrieben, das für die Nutzung von Microsoft Intune in einer eigenständigen, nur für die Cloud bestimmten Konfiguration ohne lokale Server unterstützt wird.

## Dokumentation zur Hybrid-Identität
Informationen zur konzeptuellen und verfahrensorientierten Planung, Bereitstellung und Verwaltung sind hilfreich, wenn Sie Ihre Lösung für die Verwaltung mobiler Geräte implementieren:

- [Microsoft System Center](https://technet.microsoft.com/de-DE/library/cc507089.aspx)-Lösungen können beim Sammeln und Zusammenfassen von Wissen zu Infrastruktur, Richtlinien, Prozessen und bewährten Methoden hilfreich sein, damit Ihr IT-Personal verwaltbare Systeme erstellen und Vorgänge automatisieren kann.
- [Microsoft Intune](https://technet.microsoft.com/de-DE/library/jj676587.aspx) ist ein cloudbasierter Geräteverwaltungsdienst, der Sie beim Verwalten Ihrer Computer und mobilen Geräte und beim Schützen der Unternehmensinformationen unterstützt.
- [MDM für Office 365](https://technet.microsoft.com/de-DE/library/ms.o365.cc.devicepolicy.aspx) ermöglicht Ihnen das Verwalten und Schützen mobiler Geräte, wenn diese mit Ihrem Office 365-Unternehmen verbunden sind. Sie können MDM für Office 365 nutzen, um Sicherheitsrichtlinien und Zugriffsregeln für Geräte festzulegen und die Daten mobiler Geräte zu löschen, falls Sie verloren gehen oder gestohlen werden.
