#  Erste Schritte mit Azure Government

##  Allgemeine Richtlinien für Azure Government-Kunden

Die meisten technischen Inhalte, die derzeit verfügbar sind, gehen davon aus, dass Anwendungen für den globalen Dienst und nicht für Microsoft Azure Government entwickelt werden. Daher müssen Sie sicherstellen, dass Entwickler die wichtigsten Unterschiede für Anwendungen beachten, die in Azure Government gehostet werden.

- Zunächst gibt es Unterschiede bei den Diensten und Funktionen. Daher sind bestimmte Funktionen, die in bestimmten Regionen des globalen Diensts verfügbar sind, in Azure Government möglicherweise nicht enthalten.

- Für Funktionen in Azure Government gibt es zudem hinsichtlich der Konfiguration Unterschiede zum globalen Dienst. Aus diesem Grund sollten Sie Ihren Beispielcode, Ihre Konfigurationen und Schritte überprüfen, um sicherzustellen, dass Sie innerhalb der Azure Government Cloud Services-Umgebung erstellen und ausführen.

- Darüber hinaus wird empfohlen, die technische Dienstdokumentation zu Azure Government auf dieser Website zu lesen. Die Dokumentation enthält Anleitungen und bewährte Methoden zur Abgrenzung von Azure Government sowie zu den durch Kunden regulierte/kontrollierte Daten.

Weitere Informationen und Updates erhalten Sie, indem Sie den <a href="https://blogs.msdn.microsoft.com/azuregov/">Microsoft Azure Government-Blog</a> abonnieren.

##  Azure Government – ITAR

Die Informationen auf dieser Website richten sich an Azure Government-Kunden, für die die Regelungen der USA bezüglich des internationalen Waffenhandels (International Traffic in Arms Regulations, ITAR) gelten und die Azure Government-Dienste zum Speichern, Verarbeiten oder Übertragen regulierter Informationen nutzen möchten. Die bereitgestellten Informationen beschreiben den Funktionsumfang der Dienste von Microsoft Azure Government und enthalten allgemeine Hinweise für ITAR-Kunden. Machen Sie sich zunächst mit den Funktionen von Azure Government vertraut, und wenden Sie sich bei Fragen an Ihr Kontoteam, bevor Sie Ihrem Azure Government-Abonnement ITAR-regulierte Daten hinzufügen.

Auf der <a href="http://www.microsoft.com/de-DE/TrustCenter/Compliance/default.aspx/">Compliance-Seite von Microsoft Azure Trust Center</a> finden Sie aktuelle Informationen zu den Azure Government-Diensten, für die die ITAR gelten. Gegebenenfalls sind weitere Microsoft-Dienste verfügbar, die jedoch nicht von den Azure Government-Diensten abgedeckt werden. Azure Government-Dienste ermöglichen unter anderem die Verwendung einer Vielzahl zusätzlicher Ressourcen, Anwendungen oder Dienste, die von Drittanbietern (oder von Microsoft unter separaten Nutzungsbedingungen und Datenschutzrichtlinien) bereitgestellt werden. Die Bedingungen solcher (etwa über den Marketplace angebotenen) Zusatzangebote müssen von Ihnen eigenverantwortlich geprüft werden, um sicherzustellen, dass sie Ihre ITAR-Complianceanforderungen erfüllen.

<a href="https://azure.microsoft.com/features/gov/">Azure Government</a> ist für juristische Personen verfügbar, die Daten verarbeiten, für die bestimmte gesetzliche Vorschriften und Auflagen wie etwa ITAR gelten, und die Azure Government zur Einhaltung von Vorschriften benötigen. Azure Government-Kunden müssen sich einer Eignungsprüfung unterziehen. Im Zuge der Eignungsprüfung von Microsoft muss bestätigt werden, dass Sie ITAR-regulierte Militärprodukten herstellen, exportieren oder mit ihnen handeln. Dies wird entweder durch die Registrierung beim US-amerikanischen Außenministerium oder durch eine Bürgschaftsvereinbarung mit einer Verwaltungseinrichtung belegt, die spezifische Anforderungen an die Datenverarbeitung stellt.

Fragen zur Eignung für Azure Government können an das Kontoteam gerichtet werden.
