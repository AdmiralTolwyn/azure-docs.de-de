# <a name="azure-content-delivery-network"></a>Azure Content Delivery Network
Wenn Sie Microsoft Azure bei Betrieb über 21Vianet (Azure China 21Vianet) für Websites oder mobile Apps nutzen, die Streamingmedien, Gamingsoftware, Firmwareupdates (z.B. Smart-TVs und Elektrogeräte) oder IoT-Endpunkte (Internet of Things, Internet der Dinge) (z.B. Autos und Sensoren) umfassen, kann Content Delivery Network (CDN) für kürzere Ladezeiten, eine geringere Bandbreitennutzung und eine bessere Reaktionsfähigkeit sorgen.

Lokale Anbieter bieten den CDN-Dienst in China an. (Im Vergleich dazu bietet der [Azure CDN](https://azure.microsoft.com/documentation/articles/cdn-overview/)-Dienst für globales Azure Wahlmöglichkeiten aus Microsoft, Verizon und Akamai an.) Derzeit gibt es sechs lokale CDN-Anbieter, darunter Webluker von ChinaCache, [ChineNetCenter](http://www.chinanetcenter.com/), [Fastweb](https://www.fastweb.com.cn/), und in Zukunft weitere.

So arbeiten Sie mit dem Azure CDN-Dienst: 
- [Erhalten Sie Unterstützung von 21Vianet](/azure/china/china-resources-support), der zentralen Anlaufstelle für alle Azure-Dienste, einschließlich Azure CDN.
- Melden Sie sich am [einheitlichen CDN-Verwaltungsportal](https://www.azure.cn/documentation/articles/cdn-management-portal-how-to-use/) von Azure (nicht am globalen Azure CDN-Portal an, das Sie möglicherweise verwendet haben).
