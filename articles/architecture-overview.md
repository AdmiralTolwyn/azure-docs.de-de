# <a name="application-architecture-on-microsoft-azure"></a>Anwendungsarchitektur in Microsoft Azure
In diesem Artikel werden Ressourcen zum Erstellen von Anwendungen aufgeführt, die Microsoft Azure verwenden. Es werden auch Tools aufgeführt, mit denen Sie Softwaresysteme zeichnen und visuell beschreiben können.

## <a name="design-patterns-poster"></a>Poster mit Entwurfsmustern
Microsoft Patterns & Practices hat das Buch [Cloud Design Patterns](http://msdn.microsoft.com/library/dn568099.aspx) (Cloudentwurfsmuster) veröffentlicht, das sowohl auf MSDN als auch als PDF-Download erhältlich ist. Es ist zudem ein großformatiges Poster verfügbar, auf dem alle Muster visuell dargestellt sind.

![Patterns & Practices – Poster zu Cloudmustern](./media/architecture-overview/PnPPatternPosterThumb.jpg)


## <a name="drawing-symbol-and-icon-sets"></a>Symbolsätze
[Sehen Sie sich das Trainingsvideo zu Visio und Symbolen an](http://aka.ms/CnESymbolsVideo), und [laden Sie den Cloud- und Enterprise-Symbolsatz herunter](http://aka.ms/CnESymbols), um technische Materialien zu erstellen, die Azure, Windows Server, SQL Server und andere Produkte beschreiben. Sie können die folgenden Symbole verwenden, um sie kostenlos oder gegen eine Gebühr zur Verfügung zu stellen.  
- Architekturdiagramme 
- Schulungsmaterialien 
- Präsentationen 
- Datenblätter 
- Infografiken 
- Whitepaper 
- Bücher von Drittanbietern, wenn diese Benutzer für die Verwendung von Microsoft-Produkten schulen

Diese Symbole sind nicht für die Verwendung auf Benutzeroberflächen gedacht, Sie können jedoch die Erlaubnis dazu einholen.  Die Azure-Symbole können verwendet werden. Dies gilt allerdings nur, wenn Sie die Symbole zur Darstellung der gleichen Objekte auf die gleiche Weise wie im Azure-Portal verwenden.  Das Symbol für Azure Service Bus darf z.B. nur zur Darstellung von und den Zugriff auf Azure Service Bus-Objekte verwendet werden. Drittanbietersymbole befinden sich nicht im Besitz von Microsoft. Informieren Sie sich auf den Websites der jeweiligen Unternehmen über die Verwendung ihrer Symbole.

Die CnE-Symbole liegen im Visio-, SVG- und PNG-Format vor. Weitere Anweisungen zur einfachen Verwendung der Symbole in PowerPoint sind im Symbolsatz enthalten. Der Symbolsatz wird vierteljährlich herausgegeben und aktualisiert, wenn neue Dienste erscheinen.

Zusätzliche Symbole für Microsoft Office und zugehörige Technologien stehen in der [Microsoft Office Visio-Schablone](http://www.microsoft.com/en-us/download/details.aspx?id=35772)zur Verfügung, allerdings sind diese nicht wie der CnE-Satz für Architekturdiagramme optimiert.   

Wenn Sie spezifische Fragen haben oder uns Feedback geben möchten, senden Sie uns eine E-Mail an diese Adresse: [CnESymbols@microsoft.com](mailto:CnESymbols@microsoft.com). Ihre Meinung und Ihr Feedback interessieren uns, damit wir weiter an Verbesserungen arbeiten können.

## <a name="microsoft-solutions"></a>Microsoft-Lösungen
Microsoft veröffentlicht eine Reihe allgemeiner [Lösungsarchitekturen](http://aka.ms/azblueprints), die veranschaulichen, wie bestimmte Arten von Systemen mithilfe von Microsoft-Produkten erstellt werden.

Microsoft hat zuvor bereits eine Reihe von isometrischen 3D-Blueprints veröffentlicht, die Beispielarchitekturen zeigen. Diese Blueprints wurden durch die Lösungsarchitekturen ersetzt. Der Link zu den Blueprints führt jetzt zu den Lösungen. Wenn Sie Zugriff auf die vorherigen Blueprints benötigen, senden Sie per E-Mail eine Anfrage an [CnESymbols@microsoft.com](mailto:CnESymbols@microsoft.com) .   

Die Diagramme der Pläne und der Lösungsarchitekturen verwenden Teile des [Cloud- und Enterprise-Symbolsatzes](http://aka.ms/CnESymbols).   

![Microsoft-Architekturpläne – 3D-Diagramm](./media/architecture-overview/BluePrintThumb.jpg)

## <a name="3d-blueprint-visio-template"></a>Visio-Vorlage „3D Blueprint“
Die 3D-Versionen der nicht mehr bestehenden [Microsoft-Architekturpläne](http://aka.ms/azblueprints) wurden zunächst in einem Nicht-Microsoft-Tool erstellt. Eine Vorlage für Visio 2013 (und höher) wurde am 5. August 2015 im Rahmen eines [Zertifizierungskurses zur Microsoft-Architektur auf EDX.ORG](#microsoft-architecture-certification-course) bereitgestellt.

Die Vorlage steht auch außerhalb des Kurses zur Verfügung.

* [Sehen Sie sich zunächst das Trainingsvideo an](http://aka.ms/3dBlueprintTemplateVideo) , damit Sie wissen, was die Vorlage leisten kann.   
* Laden Sie die [Microsoft Visio-Vorlage "3D Blueprint"](http://aka.ms/3DBlueprintTemplate)
* Laden Sie den [Cloud- und Enterprise-Symbolsatz](https://www.microsoft.com/en-us/download/details.aspx?id=41937) herunter, der mit der 3D-Vorlage verwendet wird. 

Wenn Sie noch Fragen haben, die in den Trainingsmaterialien nicht beantwortet werden, oder uns Feedback geben möchten, senden Sie eine E-Mail an [CnESymbols@microsoft.com](mailto:CnESymbols@microsoft.com). Diese Vorlage befindet sich nicht mehr in der aktiven Entwicklung. Sie ist jedoch nach wie vor hilfreich und relevant, da sie jede PNG-Datei sowie den [Cloud- und Enterprise-Symbolsatz](http://aka.ms/CnESymbols) verwenden kann. Beide Ressourcen werden aktualisiert.  

![Microsoft Visio-Vorlage "3D Blueprint"](./media/architecture-overview/3DBlueprintVisioTemplate.jpg)
