# <a name="azure-cosmos-db-documentation-experiment"></a>Azure Cosmos DB-Dokumentationsexperiment

Azure Cosmos DB ist eine global verteilte Datenbank mit Unterstützung mehrerer Modelle. Da Azure Cosmos DB für so viele verschiedene Aufgaben verwendet werden kann – vom Erstellen eines Graphen oder einer Dokumentdatenbank bis hin zum Migrieren Ihrer Daten aus MongoDB, Cassandra oder Azure Table Storage – kann das Organisieren der Dokumentation eine Herausforderung sein. Wir möchten Sie daher um Ihre Hilfe bitten.

Wir testen, wie in den folgenden Screenshots dargestellt, die Organisation des Inhaltsverzeichnisse und der [Landing Page](https://docs.microsoft.com/en-us/azure/cosmos-db/) von Azure Cosmos DB und würden gerne Ihr Feedback zu den vorgeschlagenen Änderungen erhalten.

## <a name="to-load-both-views"></a>So laden Sie beide Ansichten

1. Um beide Ansichten nebeneinander zu öffnen, navigieren Sie in Ihrem bevorzugten Webbrowser zur [Landing Page](https://docs.microsoft.com/en-us/azure/cosmos-db/). 
2. Stellen Sie mithilfe der obigen Screenshots fest, welche Version Sie sehen – die ursprüngliche oder die experimentelle Version. 
3. Öffnen Sie ein Browserfenster im InPrivate- oder Inkognitomodus, laden Sie die [Landing Page](https://docs.microsoft.com/en-us/azure/cosmos-db/) im neuen Browserfenster, und überprüfen Sie dann erneut mithilfe der Screenshots, welche Version Sie sehen. 

   Falls nicht beide Ansichten angezeigt werden, schließen Sie das InPrivate- oder Inkognitofenster, öffnen Sie es erneut, und laden Sie die [Landing Page](https://docs.microsoft.com/en-us/azure/cosmos-db/) noch einmal. Möglicherweise sind einige Versuche notwendig, bis beide Versionen in separaten Fenstern geladen werden. Wenn Sie das Inkognitofenster aktualisieren, wird dadurch nicht die andere Version geladen. Sie müssen ein neues Fenster öffnen. 

4. Sie können dann die Größe der Fenster ändern, um sie nebeneinander anzuzeigen, oder zwischen den Ansichten hin und her wechseln, um sie zu vergleichen. 

5. Nachdem Sie sich mit der Organisation und dem Layout der experimentellen Benutzeroberfläche vertraut gemacht haben, klicken Sie unten auf die Schaltfläche **Umfrage zum Experiment**, um Feedback abzugeben. Senden Sie das Formular bitte spätestens bis 0:00 Uhr (Mitternacht) am 4. April 2018, damit Ihr Feedback überprüft und berücksichtigt werden kann.
