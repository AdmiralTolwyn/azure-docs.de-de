# Zugreifen auf Ihre Apps

Einer der Vorteile von RemoteApp ist, dass Sie auf Apps, die für Sie veröffentlicht wurden, auf allen Ihren Geräten zugreifen können. Sie können sogar mit der Arbeit auf einem Gerät beginnen und dann reibungslos auf einem zweiten Gerät genau an der Stelle fortfahren, an der Sie aufgehört haben. Für den Einstieg müssen Sie den entsprechenden Client für Ihr Gerät herunterladen und sich beim Dienst anmelden.

In diesem Thema sehen wir uns die Liste der derzeit unterstützten Clients an. Sie erfahren, wie diese heruntergeladen werden und wie Sie sich auf jedem dieser Clients bei RemoteApp anmelden.

## Unterstützte Clients

Sie können über die folgenden Schritte auf RemoteApp zugreifen, wenn Ihr Gerät eines dieser Betriebssysteme hat:

 - Windows 10 (Preview)
 - Windows 8.1
 - Windows 8
 - Windows 7 Service Pack 1
 - Windows RT
 - Windows Phone 8.1
 - iOS
 - Mac OS X
 - Android


 Was gilt für Thin Clients? Die folgenden Windows Embedded Thin Clients werden unterstützt:

 - Windows Embedded Standard 7 mit Service Pack 1
 - Windows Embedded POSReady7
 - Windows Embedded Thin PC
 - Windows Embedded 8.1 Industry

## Herunterladen des Clients

Unabhängig davon, welche Plattform Sie verwenden, finden Sie den Client für den Zugriff auf RemoteApp auf der Seite zum [Herunterladen des Remotedesktopclients](https://www.remoteapp.windowsazure.com/ClientDownload/AllClients.aspx).

Durch Klicken auf die verschiedenen Links wird entweder der Download direkt gestartet, oder Sie werden zur Downloadseite des Clients im App-Store der jeweiligen Plattform geleitet. Installieren Sie den Client gemäß den Anweisungen auf dem Bildschirm.

Nachdem Sie den Client auf Ihrem Gerät installiert und gestartet haben, wechseln Sie zum entsprechenden nachstehenden Abschnitt, um zu erfahren, wie Sie sich auf diesem Client bei RemoteApp anmelden.

## Android

Nachdem Sie die Microsoft-Remotedesktop-App über Google Play installiert haben, finden Sie sie in Ihrer App-Liste unter **Remotedesktop**.

1. Nach Starten der App gelangen Sie zu einem leeren Connection Center, es sei denn, Sie haben die App bereits verwendet. Für den Einstieg in Azure RemoteApp tippen Sie auf die Schaltfläche **""+""** und dann auf **Azure RemoteApp**.

![Leeres Connection Center](./media/remoteapp-clients/Android1.png)

2. Sie müssen sich mit Ihrer E-Mail-Adresse beim Dienst anmelden. Um diesen Prozess zu starten, tippen Sie auf **Erste Schritte**.

![Anmeldeaufforderung](./media/remoteapp-clients/Android2.png)

3. Geben Sie auf der nächsten Seite Ihre **E-Mail-Adresse** ein, und tippen Sie auf **Weiter**. Dies startet den Anmeldevorgang mithilfe von Azure Active Directory.

![Erste Azure Active Directory-Seite](./media/remoteapp-clients/Android3.png)

4. Folgen Sie den Anweisungen auf dem Bildschirm, und melden Sie sich mit Ihrem Microsoft-Konto (LiveID) oder Ihrer Organisations-ID an. Nach der Anmeldung wird ggf. eine Seite mit allen Einladungen angezeigt, die Sie erhalten haben. Wenn Sie sicher sind, wählen Sie die Einladungen aus, denen Sie vertrauen, und tippen Sie auf **Fertig**.

![Seite "Einladungen"](./media/remoteapp-clients/Android4.png)

5. Nach dem Akzeptieren Ihrer Einladungen wird die Liste der Apps, auf die Sie Zugriff haben, auf Ihr Gerät heruntergeladen und im Connection Center zur Verfügung gestellt. Tippen Sie auf eine der Apps, um sie zu starten und mit der Nutzung zu beginnen.

![Connection Center mit einem Feed](./media/remoteapp-clients/Android5.png)

6. Wenn Sie noch keine Einladung haben, können Sie den Dienst dennoch ausprobieren. Tippen Sie dazu auf **Zur kostenlosen Testversion wechseln**, wenn Sie aufgefordert werden.

![Eingabeaufforderung für Demofeed](./media/remoteapp-clients/Android6.png)

7. Dadurch erhalten Sie Zugriff auf verschiedene grundlegende Apps für den Einstieg in RemoteApp.

![Demofeed für Azure RemoteApp](./media/remoteapp-clients/Android7.png)

## Windows (alle unterstützten Versionen außer Windows Phone)

Der Client wird nach Abschluss der Installation automatisch gestartet. Wenn Sie jedoch später erneut einmal auf ihn zugreifen müssen, finden Sie ihn in Ihrer App-Liste unter dem Namen **Azure RemoteApp**.

1. Nach dem Start des Clients werden Sie auf der ersten Seite bei Azure RemoteApp begrüßt. Um fortzufahren, klicken Sie auf **Erste Schritte**.

![Seite "Willkommen" des Azure RemoteApp-Clients](./media/remoteapp-clients/Windows1.png)

2. Auf der nächsten Seite beginnt der Anmeldeprozess für Azure RemoteApp mithilfe von Azure Active Directory. Dieser Prozess sollte Ihnen bekannt vorkommen, wenn Sie in der Vergangenheit Microsoft-Dienste verwendet haben. Starten Sie mit der Eingabe Ihrer **E-Mail-Adresse**, und klicken Sie auf **Weiter**.

![Erste Azure Active Directory-Eingabeaufforderung](./media/remoteapp-clients/Windows2.png)

3. Folgen Sie den Anweisungen auf dem Bildschirm, und melden Sie sich mit Ihrem Microsoft-Konto (LiveID) oder Ihrer Organisations-ID an. Nach der Anmeldung wird ggf. eine Seite mit allen Einladungen angezeigt, die Sie erhalten haben. Wenn Sie sicher sind, wählen Sie die Einladungen aus, denen Sie vertrauen, und klicken Sie auf **Fertig**.

![Seite "Einladungen" des Azure RemoteApp-Clients](./media/remoteapp-clients/Windows3.png)

4. Nach dem Akzeptieren Ihrer Einladungen wird die Liste der Apps, auf die Sie Zugriff haben, auf Ihr Gerät heruntergeladen und im Connection Center zur Verfügung gestellt. Doppelklicken Sie auf eine der Apps, um sie zu starten und mit der Nutzung zu beginnen.

![Connection Center des Azure RemoteApp-Clients](./media/remoteapp-clients/Windows4.png)

5. Wenn Ihnen noch niemand eine Einladung gesendet hat, seien Sie unbesorgt. Wir helfen Ihnen! Sie haben Zugriff auf eine Demosammlung, damit Sie den Dienst testen können.

![Demofeed für Azure RemoteApp](./media/remoteapp-clients/Windows5.png)
