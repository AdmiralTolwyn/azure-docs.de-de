# <a name="stay-connected-with-azure-iot-central-preview-features"></a>Aufrechterhalten der Gerätekonnektivität in Azure IoT Central (Previewfunktionen)

[!INCLUDE [iot-central-pnp-original](../../../includes/iot-central-pnp-original-note.md)]

Dieser Artikel bietet eine Übersicht darüber, wie Sie mithilfe von Azure IoT Central die laufende Konnektivität und Integrität Ihrer Geräte sicherstellen können.

Für jede IoT-Lösung, die auf den Betrieb im gewünschten Umfang ausgelegt ist, ist ein gut strukturierter Ansatz hinsichtlich der Geräteverwaltung unverzichtbar. Es reicht nicht aus, Geräte einfach mit der Cloud zu „verbinden“. Stattdessen benötigen Sie einen Ansatz, mit dem Sie die Konnektivität und Integrität Ihrer Lösung laufend sicherstellen, während diese sich weiterentwickelt, wächst und altert. Azure IoT Central bietet die nötigen Funktionen, mit denen Sie sicherstellen können, dass Geräte in Ihrer IoT-Lösung über den gesamten Anwendungslebenszyklus optimal verwaltet werden.

## <a name="dashboards"></a>Dashboards 
Integrierte [Dashboards](howto-manage-devices.md#import-devices) bieten eine anpassbare Oberfläche zum Überwachen von Integrität und Telemetrie der Geräte. Beginnen Sie mit einem vorgefertigten Dashboard aus einer [Anwendungsvorlage](howto-use-app-templates.md), oder erstellen Sie eigene Dashboards, die auf die Anforderungen Ihrer Anwendung zugeschnitten sind. Dashboards können für alle Benutzer in Ihrer Anwendung freigegeben oder als privat behandelt werden.

## <a name="rules-and-actions"></a>Regeln und Aktionen 
Erkennen Sie rasch Geräte, für die Sie Maßnahmen ergreifen müssen, indem Sie [benutzerdefinierte Regeln](tutorial-create-telemetry-rules.md) erstellen, die auf Gerätezustand und Telemetrie basieren. Konfigurieren Sie Aktionen, um die zuständigen Personen zu benachrichtigen, sodass Korrekturmaßnahmen zeitnah erfolgen.

## <a name="jobs"></a>Jobs 
Mit [Aufträgen](../core/howto-run-a-job.md?toc=/azure/iot-central/preview/toc.json&bc=/azure/iot-central/preview/breadcrumb/toc.json) können Sie Einzel- oder Massenaktualisierungen von Geräteeigenschaften, Einstellungen und Befehlen ausführen. 

## <a name="user-roles-and-permissions"></a>Benutzerrollen und -berechtigungen
Mit [Rollen und Berechtigungen](howto-manage-users-roles.md) können Sie die Erfahrung für jeden Benutzer individuell anpassen. Erstellen Sie Rollen direkt über die Benutzeroberfläche, und weisen Sie einfach die entsprechenden Berechtigungen zu.
