<a name="seller-insights-definitions"></a>Seller Insights – Definitionen
=======================

Die folgende Tabelle enthält Definitionen für viele der in Seller Insights verwendeten Begriffe.

|  **Begriff**                                        |  **Definition**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| Azure-Lizenztyp                               | Die Art der Lizenzvereinbarung, über die Kunden Azure erwerben. Wird auch als Kanal bezeichnet.                                                  |
| Azure-Lizenztyp: Cloud Solution Provider      | Der Endkunde erwirbt Azure und Ihr Marketplace-Angebot über seinen Cloud Solution Provider, der als Ihr Handelspartner fungiert.                 |
| Azure-Lizenztyp: Enterprise                   | Der Endkunde erwirbt Azure und Ihr Marketplace-Angebot über ein Enterprise Agreement, das direkt mit Microsoft abgeschlossen wurde.                  |
| Azure-Lizenztyp: Enterprise über Handelspartner  | Der Endkunde erwirbt Azure und Ihr Marketplace-Angebot über einen Handelspartner, der sein Enterprise Agreement über Microsoft bereitstellt.     |
| Azure-Lizenztyp: Nutzungsbasierte Bezahlung                | Der Endkunde erwirbt Azure und Ihr Marketplace-Angebot über eine Vereinbarung mit nutzungsbasierter Bezahlung (Pay As You Go, PAYG), die direkt mit Microsoft abgeschlossen wurde.                |
| Azure-Abonnement-GUID                          | Die GUID (Globally Unique Identifier, eindeutiger Bezeichner) des Azure-Abonnements, über das der Kunde Ihr Marketplace-Angebot erworben hat.                     |
| Abrechnungsbetrag (CC)                               | Der dem Kunden in Rechnung gestellte Betrag, in der Abrechnungswährung des Kunden (Customer Currency, CC).                                                                 |
| Abrechnungsbetrag (PC)                               | Der dem Kunden in Rechnung gestellte Betrag, in der *Auszahlungswährung* (Payout Currency, PC).                                                                      |
| Abrechnungsdatum                                      | Das Datum, an dem die Gebühren des Kunden berechnet wurden, normalerweise sofort nach dem Nutzungszeitraum.                                             |
| Cloudinstanzname                              | Die Microsoft Cloud, in der die Bereitstellung eines virtuellen Computers erfolgt ist. (Azure Government: Microsoft Cloud-Instanz für die US-Regierung; Azure China: Microsoft Cloud-Instanz in China; Azure Deutschland: Microsoft Cloud-Instanz in Deutschland; Azure Global: Microsoft Cloud-Instanzen für alle anderen globalen Standorte)                                                          |
| Nutzung nach benutzerdefinierten Verbrauchseinheiten                               | Gemessene Verbrauchseinheiten in benutzerdefinierten Volumenangeboten.                                                                                        |
| Kunde                                         | Jeder Azure-Kunde oder -Endbenutzer, der ein über den Marketplace veröffentlichtes Angebot erwirbt, verwendet oder anzeigt. Kunden können über die eindeutige *Azure-Abonnement-GUID* identifiziert werden.                                                                                                                                                                        |
| Kundenort                                    | Der Name des vom Endkunden angegebenen Orts.                                                                                                  |
| Kommunikationssprache des Kunden                  | Die vom Kunden für die Kommunikation bevorzugte Sprache.                                                                                    |
| Firmenname des Kunden                            | Der vom Endkunden angegebene Firmenname.                                                                                               |
| Land des Kunden                                 | Der vom Endkunden angegebene Ländername.                                                                                               |
| Währung des Kunden (Customer Currency, CC)                           | Die vom Kunden für Preise und Abrechnungen bevorzugte Währung.                                                                              |
| E-Mail-Adresse des Kunden                                   | Die vom Endkunden angegebene E-Mail-Adresse.                                                                                              |
| Vorname des Kunden                              | Der Vorname des Kunden.                                                                                                              |
| Nachname des Kunden                               | Der Nachname des Kunden.                                                                                                               |
| Zahlungsart des Kunden                            | Das vom Kunden verwendete Zahlungsmittel.                                                                                         |
| Postleitzahl des Kunden                             | Die vom Endkunden angegebene Postleitzahl.                                                                                                |
| Bundesland/Kanton des Kunden                                   | Das Bundesland oder der Kanton des Endkunden.                                                                                                      |
| Erwerbsdatum                                    | Das erste Datum, an dem mit dem Azure-Abonnement ein von Ihnen veröffentlichtes Angebot erworben wurde.                                                                  |
| Verlustdatum                                        | Das Datum, an dem alle von Ihnen veröffentlichten Angebote von dem Azure-Abonnement gekündigt wurden.                                                                     |
| Geschätzte erweiterte Gebühren (CC)                   | Die geschätzten erweiterten Gebühren für die Menge der Verbrauchseinheiten für eine bestimmte SKU (in der Währung des Kunden). Dieser Wert kann aufgrund von Rundungs- oder Kürzungsfehlern nicht exakt sein.                                                                                                                                                                                           |
| Geschätzte erweiterte Gebühren (PC)                   | Die geschätzten erweiterten Gebühren für die Menge der Verbrauchseinheiten für eine bestimmte SKU basierend auf dem Wechselkurs an dem Datum, an dem die Nutzung berechnet wird (in der Währung des Herausgebers). Dieser Wert kann aufgrund von Rundungs- oder Kürzungsfehlern nicht exakt sein.                                                                                                                |
| Geschätzte Auszahlung (PC)                            | Die geschätzte Zahlung für die Menge der Verbrauchseinheiten für eine bestimmte SKU basierend auf dem Wechselkurs an dem Datum, an dem die Nutzung berechnet wird (in der Währung des Herausgebers). Dieser Wert kann aufgrund von Rundungs- oder Kürzungsfehlern nicht exakt sein.                                                                                                                |
| Geschätzter Preis (PC)                             | Der geschätzte Preis für eine Verbrauchseinheit für eine bestimmte SKU basierend auf dem Wechselkurs an dem Datum, an dem die Nutzung berechnet wird (in der Währung des Herausgebers). Dieser Wert kann aufgrund von Rundungs- oder Kürzungsfehlern nicht exakt sein.                                                                                                                                            |
| Endgültiger Inkassostatus                          | Der letzte Status des Abrechnungs- und Inkassozyklus für eine bestimmte Gebühr. Mögliche Status: *Inkasso in Bearbeitung*, *Eingezogen*, *Ausbezahlt*, *Erstattung* oder *Abschreibung*.                                                                                                                                                                                      |
| Endgültiger Inkassostatus: Eingezogen               | Die Gebühr wurde eingezogen.                                                                                                                    |
| Endgültiger Inkassostatus: Inkasso in Bearbeitung  | Die Gebühr wurde noch nicht vom Kunden eingezogen. In der Microsoft Azure-Abrechnung steht die Abrechnung oder der Einzug für den Kunden noch aus.             |
| Endgültiger Inkassostatus: Erstattung                  | Die Gebühr wurde erstattet (entweder die gesamte Gebühr oder ein Teilbetrag).                                                                      |
| Endgültiger Inkassostatus: Abschreibung               | Die Gebühr wurde als Forderungsausfall abgeschrieben.                                                                                                      |
| IsNew-Kunde                                   | Der Kunde hat diese *SKU* innerhalb des Kalendermonats zum ersten Mal bereitgestellt.                                                               |
| Markt                                           | Der vom Endkunden angegebene Ländername.                                                                                               |
| Marketplace-Lizenztyp                         | Die Abrechnungsmethode für das Marketplace-Angebot.                                                                                                 |
| Marketplace-Lizenztyp: Abrechnung über Azure   | Microsoft ist Ihr Vertreter für dieses Marketplace-Angebot und erstellt Abrechnungen für Kunden in Ihrem Auftrag. (Entweder nutzungsbasierte Bezahlung per Kreditkarte oder Firmenrechnung)       |
| Marketplace-Lizenztyp: Bring Your Own License (BYOL) | Für die Bereitstellung des virtuellen Computers ist ein vom Kunden bereitgestellter Lizenzschlüssel erforderlich. Microsoft stellt Kunden die Nutzung dieser Marketplace-Angebote nicht in Rechnung. |
| Marketplace-Lizenztyp: Kostenlos                   | Das Marketplace-Angebot ist für alle Benutzer als kostenlos konfiguriert. Microsoft stellt Kunden die Nutzung dieses Marketplace-Angebots nicht in Rechnung.    |
| Marketplace-Lizenztyp: Microsoft als Handelspartner  | Microsoft ist Ihr Handelspartner für dieses Marketplace-Angebot.                                                                                       |
| Marketplace-Bestellung                                | Für virtuelle Computer stellt eine Marketplace-Bestellung die Bereitstellung eines oder vieler virtueller Computer dar, die mit einer einzelnen *SKU* in einem eindeutigen Azure-Abonnement verbunden sind. Eine einzelne Marketplace-Bestellung kann viele Bereitstellungen mit variablen Kerngrößen darstellen. Für verwaltete Anwendungen und Entwicklerdienste stellt eine Marketplace-Bestellung den Kauf einer einzelnen *SKU* über eine *Azure-Abonnement-GUID* dar.           |
| Microsoft-Gebühr (CC)                               | Die Microsoft-Gebühr für die Transaktion in der Währung des Kunden.                                                                                 |
| Normalisierter Verbrauch                                 | Die normalisierte Nutzungsdauer in Stunden zur Berücksichtigung bei der Anzahl der verwendeten VM-Kerne: [Anzahl der VM-Kerne] x [Stunden der Rohnutzung]. Für als „SHAREDCORE“ festlegte virtuelle Computer wird der Multiplikator 1/6 (oder 0,1666) für [Anzahl der VM-Kerne] verwendet.                                                                                                                                       |
| Angebotsname                                       | Der Name des Marketplace-Angebots.                                                                                                           |
| Angebotstyp                                       | Der Typ der Lösung.                                                                                                                        |
| Angebotstyp: Benutzerdefinierte Volumenlösung                | Der Kunde hat Ihre benutzerdefinierte Volumenlösung aus dem Marketplace bereitgestellt.                                                                     |
| Angebotstyp: Verwaltete Anwendung                  | Der Kunde hat ein Abonnement für Ihre verwaltete Dienstanwendung erworben.                                                                   |
| Angebotstyp: Einzelner virtueller Computer                            | Der Kunde hat ein Image durch Auswählen eines einzelnen virtuellen Computers bereitgestellt.                                                                                     |
| Angebotstyp: Lösungsvorlage                    | Der Kunde hat Ihre Lösungsvorlage bereitgestellt.                                                                                                |
| Stornierungsdatum der Bestellung                                | Das Datum, an dem die Marketplace-Bestellung storniert wurde.                                                                                                 |
| OrderCount                                      | Die Anzahl der (aktiven und stornierten) Bestellungen für die spezifische *Azure-Abonnement-GUID* für Ihre Angebote.                               |
| Kaufdatum der Bestellung                              | Das Datum, an dem die *Marketplace-Bestellung* erstellt wurde. Für virtuelle Computer ist dies das erste Datum, an dem Ihr Image im Azure-Abonnement des Kunden bereitgestellt wurde. Alle nachfolgenden Bereitstellungen desselben Images für das gleiche Azure-Abonnement gelten als eine Bestellung.                                                                                                       |
| Bestellstatus                                     | Der Status einer Marketplace-Bestellung zu dem Zeitpunkt, zu dem die Daten zuletzt aktualisiert wurden.                                                                   |
| Bestellstatus: Aktiv                             | Bei verwalteten Anwendungen ist die Bestellung aktiv. Bei virtuellen Computern verfügt der Kunde über mindestens eine Bereitstellung des Images in seinem Azure-Abonnement.   |
| Bestellstatus: Storniert                          | Bei verwalteten Anwendungen wurde die Bestellung storniert. Bei virtuellen Computern hat der Kunde alle Bereitstellungen der *SKU* aus seinem Azure-Abonnement gelöscht.        |
| OrderID                                          | Der eindeutige Bezeichner der Bestellung. Bei verwalteten Anwendungen hat der Kunde ein Monatsabonnement für Ihren Dienst erworben. Bei virtuellen Computern hat der Kunde Ihr Image bereitgestellt. Alle nachfolgenden Bereitstellungen desselben Images für das gleiche Azure-Abonnement gelten als eine Bestellung.                                                                       |
| Zahlungsart: Karte                               | Der Kunde zahlt seine Marketplace-Gebühren mit einer Kreditkarte.                                                                          |
| Zahlungsart: Rechnung                            | Der Kunde zahlt seine Marketplace-Gebühren per Rechnung.                                                                                 |
| Auszahlungsbetrag (PC)                               | Der an Sie ausgezahlte Betrag, in Ihrer bevorzugten *Auszahlungswährung* (Payout Currency, PC).                                                                            |
| Auszahlungswährung (PC)                             | Die für Ihre Auszahlungen verwendete Währung.                                                                                                          |
| Auszahlungsdatum                                      | Das Datum, an dem die Zahlungsanforderung von Microsoft an Ihre Bank gesendet wurde.                                                                           |
| Auszahlungsstatus                                    | Gibt den Status der Transaktion im Auszahlungszyklus an: *Ausbezahlt*, *Bevorstehende Auszahlung* oder *Nicht auszahlungsbereit*.                        |
| Auszahlungsstatus: Nicht auszahlungsbereit              | Die Transaktion ist nicht bereit für die Auszahlung. (Weitere Informationen dazu unter *Endgültiger Inkassostatus*)                                                    |
| Auszahlungsstatus: Ausbezahlt                          | Die Transaktion war in einer früheren Auszahlungsberechnung enthalten. Positive Werte werden bezahlt, und negative Werte werden mit dem fälligen Gesamtbetrag verrechnet. |
| Auszahlungsstatus: Bevorstehende Auszahlung                   | Die Transaktion ist bereit für die Auszahlung und wird in die nächste verfügbare Auszahlungsberechnung eingefügt.                                           |
| Vorschau-SKU                                      | Sie haben die *SKU* als „Vorschauversion“ markiert, sodass dieses Image nur in den in Ihrer Whitelist enthaltenen Azure-Abonnements bereitgestellt und verwendet werden kann.                       |
| Preis (CC)                                       | Der Preis für eine Verbrauchseinheit einer bestimmten SKU (in der Währung des Kunden).                                                                  |
| Zustimmung zur Kontaktaufnahme zu Werbezwecken                       | Gibt an, ob der Kunde proaktiv der Teilnahme an der Kontaktaufnahme zu Werbezwecken durch Herausgeber zugestimmt hat. Derzeit wird diese Option für Kunden nicht angezeigt. Daher ist allgemein „Nein“ angegeben. Sobald diese Funktion bereitgestellt wird, wird eine entsprechende Aktualisierung vorgenommen.                                                                                          |
| Währung des Herausgebers (Publisher Currency, PC)                          | Die vom Herausgeber für Auszahlungen bevorzugte Währung.                                                                                          |
| Rohnutzung                                        | Die Nutzungsdauer in Stunden für Ihr Marketplace-Angebot.                                                                                                      |
| E-Mail-Adresse des Handelspartners                                   | Die E-Mail-Adresse des am Verkauf an den Endkunden beteiligten Handelspartners.                                                                  |
| Name des Handelspartners                                    | Der Name des Microsoft-Handelspartners für den Endkunden.                                                                                |
| Ressourcen-URI                                     | Ein eindeutiger Bezeichner für einzelne virtuelle Computer oder Bereitstellungen von Entwicklerdiensten.                                                                    |
| SKU                                              | Der bei der Veröffentlichung definierte *SKU*-Name. Ein Angebot kann mehrere *SKUs* enthalten, aber eine *SKU* kann nur einem einzigen Angebot zugeordnet werden.              |
| SKU-Abrechnungstyp                                 | Die Abrechnungsmethode für die *SKU*.                                                                                                             |
| SKU-Abrechnungstyp: Bring Your Own License (BYOL)                           | Für die Bereitstellung des virtuellen Computers ist ein vom Kunden bereitgestellter Lizenzschlüssel erforderlich. Microsoft stellt Kunden die Nutzung dieser Marketplace-Angebote nicht in Rechnung.   |
| SKU-Abrechnungstyp: Kostenlos                           | Die *SKU* ist für alle Benutzer als kostenlos konfiguriert. Microsoft stellt Kunden die Nutzung dieser *SKU* nicht in Rechnung.                            |
| SKU-Abrechnungstyp: Microsoft als Handelspartner          | Microsoft ist Ihr Handelspartner für diese *SKU*.                                                                                                   |
| SKU-Abrechnungstyp: Bezahlt                           | Microsoft ist Ihr Vertreter für diese *SKU* und erstellt Abrechnungen für Kunden in Ihrem Auftrag. (Entweder nutzungsbasierte Bezahlung per Kreditkarte oder Firmenrechnung)                   |
| SKU-Abrechnungstyp: Kostenlose Testversion                          | Für den Kunden gilt noch der Testzeitraum. Wird in „Bezahlt“ geändert, wenn der Kunde die Testversion nicht kündigt oder löscht.                                         |
| Steuerbetrag (CC)                                  | Der für die Abrechnung des Kunden geltend gemachte Steuerbetrag in der *Währung des Kunden (CC)*.                                                               |
| Transaktionsdatum                                 | Das in Ihren Auszahlungsberichten verzeichnete Datum der Transaktion.                                                                               |
| Transaktionstyp                                 | Der Typ der Transaktion, die protokolliert wird.                                                                                              |
| Transaktionstyp: Gebühr                         | Die Transaktion ist ein positiver Wert, der den dem Kunden in Rechnung gestellten Betrag angibt.                                                         |
| Transaktionstyp: Erstattung für Kunden                | Die Gebühr des Kunden wurde erstattet. Diese Transaktion ist ein negativer Wert, der dem Betrag der positiven Gebühr des Kunden entspricht. Sie können den entsprechenden zuvor bezahlten positiven Wert identifizieren, indem Sie die Transaktion mit dem gleichen *Abrechnungsdatum* und *Transaktionstyp* = „Gebühr“ und *Endgültiger Inkassostatus* = „Erstattung“ identifizieren.                   |
| Transaktionstyp: Auszahlungsanpassung              | Die Transaktion stellt eine positive oder negative Anpassung dar, die von Microsoft auf Ihren Saldo angewandt wird und erstellt wurde, um einen früheren Abrechnungs- oder Auszahlungsfehler zu berücksichtigen.                                                                                                                                                                                            |
| Transaktionstyp: Abschreibung                      | Die Gebühr des Kunden wurde als Forderungsausfall abgeschrieben. Diese Transaktion ist ein negativer Wert, der dem Betrag der positiven Gebühr des Kunden entspricht. Sie können den entsprechenden zuvor bezahlten positiven Wert identifizieren, indem Sie die Transaktion mit dem gleichen *Abrechnungsdatum* und *Transaktionstyp* = „Gebühr“ und *Endgültiger Inkassostatus* = „Abschreibung“ identifizieren. |
| Transaktionstyp: Stornierung der Abschreibung             | Die Transaktion ist ein positiver Wert und stellt die Stornierung einer zuvor abgeschriebenen Transaktion dar.                                         |
| Enddatum der Testversion                                   | Das Datum, an dem der Testzeitraum für die Bestellung endet oder beendet ist.                                                                              |
| Verwendung                                            | Die protokollierte Nutzung der *SKU* durch den Kunden. Bei VM-Images stellen Nutzungseinträge die Nutzung für den protokollierten Zeitraum für die jeweilige VM-Größe und *SKU* dar.   |
| Datum für Nutzungsende                                   | Das Enddatum des protokollierten Nutzungszeitraums.                                                                                             |
| Usage Date                                       | Das Datum, an dem die Nutzung durch den Kunden erfolgt ist.                                                                                                             |
| Nutzungsreferenz                                  | Der Bezeichner für einen oder mehrere Tage der Nutzung einer bestimmten SKU durch den Kunden, der einem Eintrag im Auszahlungsbericht zugeordnet ist.                         |
| Datum für Nutzungsbeginn                                 | Das Startdatum des protokollierten Nutzungszeitraums.                                                                                           |
| Nutzungstyp                                       | Eine Beschreibung der gemessenen Nutzung. (*Normalisierter Verbrauch* oder *Rohnutzung*)                                                               |
| Verbrauchseinheiten                                      | Die Maßeinheit für die angegebene Nutzung. Virtuelle Computer werden immer in stündlichen Maßeinheiten gemessen.                                      |
| Größe des virtuellen Computers                                          | Stellt die mit dem Azure-Angebot abgestimmte Hardwaregröße des virtuellen Computers dar. Beispiele hierfür sind `Basic_A0`, `Standard_A11`, `Standard_D12` und `Standard_G4`.   |
|  |  |
